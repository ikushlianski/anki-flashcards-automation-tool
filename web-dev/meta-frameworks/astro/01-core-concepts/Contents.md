# Core Concepts

## What This Section Covers

**Islands Architecture**
- Understanding the islands pattern and its benefits
- How Astro renders static HTML by default with selective hydration
- When and why to use client-side JavaScript islands

**Server-First Design**
- Moving expensive rendering off client devices
- Zero JavaScript by default approach
- Performance optimization through minimal client-side code

**Framework Agnostic**
- How Astro works with multiple UI frameworks (React, Vue, Svelte, Solid, etc.)
- Mixing different frameworks in the same project
- When to use framework components vs native Astro components

**Component-Based Architecture**
- Understanding Astro's component model
- Similarities and differences from other frameworks like Next.js
- Static vs interactive components

**Build vs Runtime**
- How Astro processes pages at build time
- Understanding the distinction between build-time and runtime code
